Try lein-try

Apr 29 2015

In this fast-growing Clojure community there are new libraries popping up all the time. I often want to just tinker around with a few libraries, and it’s a pain creating a temporary project and setting it up with the appropriate dependencies before I can start using the library in the REPL. The REPL is a great tool for playing around with new libraries and sometimes you’ll learn enough inside the REPL and won’t have to add any code to the project at all. Wouldn’t it be better if we could just skip the whole temporary project step and use the REPL from the very beginning?

Well with lein-try, you can!

Give it a go:

1

lein try prismatic/plumbing "0.4.2"

Then once in the REPL you can just use a namespace:

1
2
3
4
5
6

(use 'prismatic.core)
;;=> nil
(def my-fnk (fnk [a b {c 4} d {e 2}] (+ a b c d e)))
;;=> #'user/my-fnk
(my-fnk {:a 1 :b 2 :d 3})
;;=> 12

You can leave out the version number. This will simply pull the latest version from Clojars.

1

lein try prismatic/plumbing

And you can supply multiple libraries:

1

lein try prismatic/plumbing org.flatland/useful

Let’s require and name a namespace from each:

1
2
3
4
5
6

(require '[useful.map :as m1])
;;=> nil
(require '[plumbing.map :as m2])
;;=> nil
(m1/map-keys {1 "a" 2 "b"} inc)
;;=> {2 "a" 3 "b"}

That’s it! Give it a go!
